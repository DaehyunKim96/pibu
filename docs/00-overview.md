# 00 · 서비스 개요 & 기획

> 작성: pibu 초안 기획 단계 / 최종 갱신 2026-06-20

## 1. 한 줄 정의

**pibu** = "내 피부가 왜 예민한지부터 성분으로 설명해 주는, 민감·트러블 피부 전용 제품 큐레이션."

## 2. 문제 정의 (왜 만드는가)

- 시중 스킨케어는 **마케팅 문구 중심**이라, 예민한 사람은 "이게 나한테 맞는지"를 판단하기 어렵다.
- 전성분은 공개돼 있지만 일반 사용자가 **성분의 의미를 해석**하기 어렵다.
- 민감·트러블 피부는 잘못된 제품 하나로 며칠을 고생한다 → **실패 비용이 크다**.
- 기존 성분 분석 서비스(화해 등)는 "위험도 등급" 중심이라, **내 피부 타입과 연결된 맥락**이 약하다.

## 3. pibu의 차별점 (기획 가설)

1. **성분 → 특성 → 키워드 → 타입**을 하나의 사슬로 연결한다.
   - 단순히 "이 성분 위험" 이 아니라 "이 성분 때문에 이 제품은 이런 특성을 가진다"를 설명.
2. **민감/트러블 피부를 1순위 페르소나**로 삼는다. (범용 뷰티앱과의 포지셔닝 차별)
3. **근거(논문)를 같이 보여준다.** 신뢰가 핵심 자산.
4. **'덜어내기' 철학** — 더 많이 파는 게 아니라, 안 맞는 걸 거르게 돕는다.

## 4. 범위 (이번 초안에 포함 / 제외)

## 6. 정보 구조(IA)

```
홈
├─ 제품 (목록·필터) ─→ 제품 상세 (자세히 보기)
├─ 피부 타입 (진단 → 결과 → 추천 제품)
├─ 성분 사전 (카테고리별 + 논문 근거)
└─ 트러블 가이드 (피부과학 원칙)
```

## 7. 데이터 모델 한눈에

- `Ingredient` : 작용·근거·민감 적합도(1~5)·부여 특성·궁합
- `Product` : 핵심 성분(→Ingredient id)·특성·키워드·적합 타입·highlights(성분→효과)
- `SkinType` : 4축 분류·추천/주의 성분·케어 팁
- 세 데이터가 **id로 서로 연결**되어, 한 곳을 고치면 전체가 일관되게 갱신됨.

## 8. 성공 지표 (제안)

- 진단 완료율, 진단→제품 상세 전환율
- "자세히 보기" 도달률 (성분 설명이 읽히는가)
- 무향/타입 필터 사용률 (개인화 수요 검증)
