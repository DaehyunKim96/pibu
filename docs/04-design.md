## 레퍼런스 리서치 (피부/뷰티 서비스)

| 서비스 | 배울 점 | pibu 적용 |
| --- | --- | --- |
| **올리브영** | 카테고리 필터·카드 그리드의 명료함 | 제품 그리드 + 다중 필터 |
| **화해(Hwahae)** | 성분 분석을 일반인이 이해하게 | 성분 사전 + 적합도 시각화 |
| **CeraVe/La Roche-Posay (더마)** | 차분·임상적 신뢰감, 절제된 색 | 세이지 그린 + 뉴트럴 팔레트 |
| **Soko Glam / Sephora** | 진단형 추천(스킨 퀴즈) | 4축 진단 → 타입 결과 |
| **Typology (FR)** | 투명한 성분·근거 노출, 미니멀 | "왜 이런 특성?" 근거 카드 |

## 디자인 원칙 (예민한 피부 사용자를 위해)

1. **진정되는 비주얼** — 자극적인 고채도 대신 **세이지 그린 + 따뜻한 모래색**.
   민감 피부 사용자의 심리적 안정과 브랜드 톤(피부를 달랜다)을 일치.
2. **신뢰 = 투명성** — 효과뿐 아니라 **주의 성분/근거**를 같은 비중으로 노출.
   과장된 "기적" 카피 금지. 면책 문구 상시 노출.
3. **읽기 쉬운 위계** — 카드 → 특성 칩 → 키워드 → "자세히 보기"로 정보 깊이를 단계화.
   한 화면에 다 쏟지 않고 점진적으로 공개(progressive disclosure).
4. **접근성** — 충분한 본문 대비, 16px+ 본문, 한글 가독 폰트(Noto Sans KR + Gowun Dodum),
   반응형(데스크탑 3열 → 태블릿 2열 → 모바일 1열).

## 디자인 토큰 (요약)

- **색**: `--green-600 #5b8a72`(주조) · `--sand-100 #f7f3ec`(배경) · `--warn #c0584f`(주의)
- 성분 카테고리별 색: 진정=그린, 장벽=블루, 활성=브라운, 항산화=퍼플, 주의=레드
  → 성분 사전·제품 상세에서 **색으로 카테고리를 직관 인지**.
- **라운드**: 카드 16px / 큰 영역 24px / 칩 pill — 부드러운 인상.
- **그림자**: 3단계(sm/md/lg)로 깊이감, 과하지 않게.
- **타이포**: 제목 Gowun Dodum(둥근 명조 느낌), 본문 Noto Sans KR.

## 핵심 화면 의도

- **홈**: 히어로에서 "왜 예민한지부터 찾아준다"는 약속 → 진단 CTA를 최상단에.
- **제품 상세**: "🔬 왜 이런 특성을 가질까?" 섹션이 페이지의 주인공.
  성분 카테고리 색 보더로 시각적 근거를 줌.
- **진단 결과**: 타입 색으로 배너를 채우고, "챙길 성분/조심할 성분"을
  초록 vs 빨강 2열로 즉각 대비.

## 의도적으로 하지 않은 것 (v0.1)

- 화려한 애니메이션/패럴랙스 → 신뢰감 우선, 산만함 배제.
- 다크모드 → v0.2 후보 (토큰 기반이라 확장 쉬움).
- 외부 UI 프레임워크 → 번들 가볍게, 디자인 완전 제어.

## 다음 디자인 과제 (`05-roadmap`와 연계)

- 제품 비교(2~3개 나란히), 성분 충돌 경고 배지, 진단 결과 공유 카드(SNS).
- 사용자 테스트로 "자세히 보기" 도달률 측정 → 정보 위계 재조정.
